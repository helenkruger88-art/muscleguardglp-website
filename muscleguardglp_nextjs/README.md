# Muscle Guard — muscleguardglp.com (Next.js on Vercel)

A rebuild of muscleguardglp.com per the Build Brief dated 10 June 2026.

This site is content. Every page is statically generated (SSG). All primary content is present in the initial HTML so AI crawlers (which do not run JavaScript) can read it. No third-party advertising trackers. The hero is the cited Research Hub at `/research/`; `/for-doctors/` is a footer-only support page.

## Stack

- **Next.js 15** App Router, SSG everywhere
- **React 19**, TypeScript
- **Tailwind CSS** with brand tokens ported from the existing `styles.css`
- **MDX** for research articles via `next-mdx-remote/rsc` and `gray-matter`
- **Plausible** for cookieless analytics (loaded in `src/app/layout.tsx`)
- Deploys to **Vercel** out of the box

## Local development

```bash
npm install
npm run dev       # http://localhost:3000
npm run build     # production build
npm run type-check
```

## Vocabulary rules (verbatim from Build Spec v2 §8)

- Use **muscle loss / lean mass loss** in all user-facing copy.
- The term **sarcopenia** appears only inside articles where the cited studies use it — never in headlines, nav, CTAs, or FAQ.
- Use **healthcare provider / your doctor** — never "prescriber" in user-facing copy.
- Use **self-tracking companion / coach** — never "medical device / diagnostic tool / clinical tool."
- The label for the in-app PDF export is **Clinical Report** (not "Doctor PDF" — terminology corrected mid-project).

If anything in this codebase drifts from the locked spine, edit `src/lib/site.ts` — it is the single source of truth for hero copy, pricing, drug list, email channels, and the disclaimer.

## Deploy to Vercel

```bash
# 1. Create a new Vercel project from this directory
vercel link

# 2. First production deploy
vercel --prod
```

Vercel auto-detects Next.js. No environment variables required for v1.

After first deploy:
1. Submit `https://muscleguardglp.com/sitemap.xml` to Google Search Console and Bing Webmaster Tools.
2. Test `https://muscleguardglp.com/robots.txt` returns 200 and lists all AI bots.
3. Test `https://muscleguardglp.com/llms.txt` returns 200 with the curated Markdown.
4. Run Google Rich Results Test on the homepage, FAQ, and one research article — expect zero errors.
5. Run PageSpeed Insights on the homepage — target CWV "Good" on mobile.
6. Test the 301 redirect map: `curl -I https://muscleguardglp.com/muscle-loss-on-glp1.html` should return `301` → `/research/glp1-and-muscle-loss/`.
